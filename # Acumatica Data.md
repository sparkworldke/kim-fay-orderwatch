I am requesting full implementation of an Acumatica integration platform with the following technical and functional requirements, along with comprehensive validation and deployment standards:

### Core Integration Requirements
1. **Customer Synchronization**: Develop a bidirectional full synchronization capability for all existing customer records between the local system and Acumatica. Implement pre-sync and post-sync data validation checks that verify 100% accuracy of all customer attributes (including billing/shipping addresses, contact information, tax IDs, and payment terms) to eliminate data discrepancies. Generate a post-sync reconciliation report that flags any failed records for manual remediation.

2. **Sales Order Synchronization**: Build a user-facing date-range selector component that restricts sales order synchronization exclusively to the user-specified start and end dates. Implement validation logic to confirm 100% of orders with creation, modification, or shipment dates falling within the selected period are captured, synced without omissions, and maintain data integrity. Mandatorily attach all core sales order attributes including order status, creation date, last modified date, shipment date, and due date to every synced record to enable enhanced visibility in the local system.

3. **Customer Category Validation & Sync**: Implement an automated background sync job that synchronizes all customer category master data from Acumatica to the local system. Add continuous validation checks to ensure category assignments for all customer records in the local system precisely match the official category mappings maintained in Acumatica. Implement structured error logging with timestamps, customer IDs, and mismatch details for any unrecognized, mismatched, or deprecated categories to enable proactive resolution.

4. **Selective Customer Order Sync with Product Association**: Develop an optional manual sync feature that allows users to select specific customers from a searchable list. Upon user initiation, trigger a targeted synchronization of all historical and active sales orders associated with the selected customers, including full line-item product details. Ensure accurate mapping and sync of all product attributes including SKUs, unit quantities, unit pricing, extended totals, discount codes, and inventory IDs to their linked sales order line items.

### General Implementation Requirements
- Implement comprehensive structured logging for all sync operations, including log entries for sync initiation, completion timestamps, record counts processed, successful records, and failed records, stored in a searchable log repository.
- Build robust error handling mechanisms to manage sync failures, including automatic retry logic for transient errors, dead-letter queue storage for permanently failed records, and automated alert notifications to system administrators for critical sync interruptions.
- Conduct full end-to-end testing of all four integration requirements to validate functionality, data accuracy, and error handling in the staging environment prior to deployment. Repeat all test cases in the production environment post-deployment to confirm production readiness and compliance with all specified requirements.
